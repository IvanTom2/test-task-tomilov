# Test Task 2

## Описание

Асинхронный API-клиент для работы с GitHub REST API. 

## Запуск

1. **Клонируйте репозиторий и перейдите в папку проекта:**

    ```bash
    git clone <repo>
    cd test-task-tomilov/2
    ```

2. **Создайте файл `.env` на основе примера `example.env` и заполните необходимые переменные окружения:**

    ```bash
    cp .env.example .env
    # Отредактируйте .env - заполните его данными подключения к GitHub
    ```

3. **Установите зависимости:**

    ```bash
    uv sync
    ```

    #### Или через Python
    ```bash
    python3 -m venv .venv
    python3 -m pip install .
    ```

4. **Запустите тесты:**

    ```bash
    uv run pytest .
    ```

    #### Или через обычный Python
    ```bash
    source .venv/bin/activate
    python3 -m pytest .
    ```

5. **Запустите проект:**

    ```bash
    uv run uvicorn main:create_app --factory
    ```

    #### Или через обычный Python
    ```bash
    source .venv/bin/activate
    python3 -m uvicorn main:create_app --factory
    ```

## Примечания
- На данный момент реализован упрощенный функционал клиента. Например, нельзя собирать бОльшее количество репозиториев на GitHub. Функционал намеренно ограничен. 
- На данный момент реализован простой рейт-лимит, который не подходит для работы в рамках распределенной системы и даже многопоточной системы (т.к. он не является потокобезопасным). 
- На данный момент реализован простой кэш запросов, который не подходит для работы в рамках распределенной системы. 


## Особенности
- Для работы необходим файл `.env` с токеном GitHub. 
- Используется aiohttp.
